# 五金销售管理系统 - 接口文档

> Base URL: `http://localhost:8080`
>
> 认证方式：登录成功后在请求头中携带 `Authorization: <token>`
>
> 统一响应格式：

```json
{
  "code": 200,
  "msg": "success",
  "data": {}
}
```

| code | 说明 |
|------|------|
| 200 | 成功 |
| 401 | 未登录或登录已过期 |
| 500 | 业务异常 / 系统异常（msg 中有具体原因） |

---

## 1. 认证模块 `/api/auth`

### 1.1 登录

- **POST** `/api/auth/login`
- **无需认证**

> 当前 Web 后台仅允许 `ADMIN` 角色登录，`SUPPLIER` 供应商账号后续走微信小程序登录入口。

**请求体：**

```json
{
  "username": "admin",
  "password": "admin123"
}
```

**响应 data：**

```json
{
  "token": "xxxx-xxxx-xxxx",
  "user": {
    "id": 1,
    "username": "admin",
    "nickname": "管理员",
    "avatar": null,
    "phone": null,
    "role": "ADMIN",
    "status": 1,
    "createTime": "2025-01-01 00:00:00",
    "updateTime": "2025-01-01 00:00:00"
  }
}
```

### 1.2 登出

- **POST** `/api/auth/logout`

**响应：** `data: null`

### 1.3 获取当前用户信息

- **GET** `/api/auth/info`

**响应 data：** 同登录返回的 `user` 对象

## 2. 商品分类 `/api/category`

### 2.1 查询全部分类

- **GET** `/api/category/list`

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| status | int | 否 | 分类状态：`0-停用 1-启用`；不传则返回全部 |

**响应 data：** `ProductCategory[]`

```json
[
  {
    "id": 1,
    "name": "电动工具",
    "sort": 1,
    "status": 1,
    "productCount": 4,
    "createTime": "2025-01-01 00:00:00"
  }
]
```

### 2.2 新增分类

- **POST** `/api/category`

```json
{ "name": "电动工具", "sort": 1, "status": 1 }
```

### 2.3 修改分类

- **PUT** `/api/category`

```json
{ "id": 1, "name": "手动工具", "sort": 2 }
```

### 2.4 启用/停用分类

- **PUT** `/api/category/{id}/status/{status}`

`status`：`0-停用 1-启用`

### 2.5 迁移分类下商品

- **PUT** `/api/category/{id}/migrate/{targetCategoryId}`

**响应 data：** `long`，表示迁移的商品数量

### 2.6 删除分类

- **DELETE** `/api/category/{id}`

说明：仅允许删除“已停用且无商品”的空分类；若分类下仍有商品，需要先迁移。

## 3. 商品管理 `/api/product`

### 3.1 分页查询

- **GET** `/api/product/page`

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| pageNum | int | 否 | 页码，默认 1 |
| pageSize | int | 否 | 每页条数，默认 10 |
| name | string | 否 | 商品名称（模糊） |
| brand | string | 否 | 品牌（模糊） |
| categoryId | long | 否 | 分类 ID |

**响应 data：** 分页对象

```json
{
  "records": [
    {
      "id": 1,
      "categoryId": 1,
      "name": "电钻",
      "brand": "博世",
      "spec": "GSB 13 RE",
      "description": "冲击钻",
      "unit": "台",
      "purchasePrice": 350.00,
      "salePrice": 500.00,
      "stock": 100,
      "restockThreshold": 10,
      "restockStatus": 0,
      "restockSupplierId": null,
      "createTime": "...",
      "updateTime": "..."
    }
  ],
  "total": 50,
  "size": 10,
  "current": 1,
  "pages": 5
}
```

### 3.2 查询详情

- **GET** `/api/product/{id}`

### 3.3 新增商品

- **POST** `/api/product`

```json
{
  "categoryId": 1,
  "name": "电钻",
  "brand": "博世",
  "spec": "GSB 13 RE",
  "description": "冲击钻",
  "unit": "台",
  "purchasePrice": 350.00,
  "salePrice": 500.00,
  "stock": 0,
  "restockThreshold": 10
}
```

### 3.4 修改商品

- **PUT** `/api/product`

> 修改商品基础信息时会保留当前补货状态，不会因为编辑名称或价格而清空“补货中”。

```json
{ "id": 1, "salePrice": 520.00 }
```

### 3.5 完成补货

- **PUT** `/api/product/restock-complete/{id}`

> 管理员确认收货后，将商品从“补货中”恢复为“正常”。

### 3.6 删除商品

- **DELETE** `/api/product/{id}`

## 6. 采购进货 `/api/purchase`

### 6.1 分页查询

- **GET** `/api/purchase/page`

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| pageNum | int | 否 | 默认 1 |
| pageSize | int | 否 | 默认 10 |
| orderNo | string | 否 | 采购单号（模糊） |
| supplierId | long | 否 | 供应商 ID |
| startDate | string | 否 | 开始日期，如 `2025-01-01` |
| endDate | string | 否 | 结束日期 |

**响应 data records：**

```json
{
  "id": 1,
  "orderNo": "PO202501011234561234",
  "supplierId": 1,
  "supplierName": "XX五金有限公司",
  "totalAmount": 10500.00,
  "paymentStatus": 0,
  "orderDate": "2025-01-01",
  "remark": "紧急采购",
  "createTime": "...",
  "updateTime": "..."
}
```

### 6.2 查询详情（含明细）

- **GET** `/api/purchase/{id}`

**响应 data：** 采购单 + `items` 明细数组

```json
{
  "id": 1,
  "orderNo": "PO...",
  "supplierName": "XX五金有限公司",
  "totalAmount": 10500.00,
  "paymentStatus": 0,
  "orderDate": "2025-01-01",
  "items": [
    {
      "id": 1,
      "orderId": 1,
      "productId": 1,
      "productName": "电钻",
      "quantity": 30,
      "price": 350.00,
      "amount": 10500.00
    }
  ]
}
```

### 6.3 创建采购单

- **POST** `/api/purchase`

> 自动生成单号、按供应商商品报价计算总金额、增加库存、记录库存流水、生成财务支出记录
>
> 约束说明：
> - 采购商品必须已经在 `supplier_product` 中维护为该供应商的供货商品
> - 采购单价格统一取 `supplier_product.supply_price`
> - 前端不允许手改采购价，后端也会忽略外部传入价格

```json
{
  "supplierId": 1,
  "orderDate": "2025-01-01",
  "remark": "紧急采购",
  "items": [
    { "productId": 1, "quantity": 30 },
    { "productId": 2, "quantity": 50 }
  ]
}
```

### 6.4 结算采购单

- **PUT** `/api/purchase/settle/{id}`

> 将采购单和关联财务记录标记为已结算

## 10. 站内消息 `/api/message`

### 10.1 分页查询

- **GET** `/api/message/page`

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| pageNum | int | 否 | 默认 1 |
| pageSize | int | 否 | 默认 10 |
| receiverId | long | 是 | 接收人用户 ID |
| isRead | int | 否 | 0-未读 1-已读 |
| type | string | 否 | GENERAL-普通消息 RESTOCK_NOTICE-补货提醒 RESTOCK_REPLY-补货回复 |

**响应 data records：**

```json
{
  "id": 1,
  "senderId": 1,
  "senderName": "管理员",
  "receiverId": 3,
  "type": "RESTOCK_NOTICE",
  "productId": 2,
  "productName": "电钻",
  "replyToMessageId": null,
  "replyToTitle": null,
  "title": "电钻 补货提醒",
  "content": "当前商品库存偏低，请尽快安排补货。",
  "isRead": 0,
  "createTime": "..."
}
```

### 10.2 发送普通消息

- **POST** `/api/message`

```json
{
  "receiverId": 3,
  "title": "月底对账提醒",
  "content": "请留意本月对账单。"
}
```

> `senderId` 由后端根据当前登录用户自动写入。

### 10.3 发送补货提醒

- **POST** `/api/message/restock`

> 发送成功后，系统会自动把对应商品改成“补货中”，并记录当前跟进供应商。

```json
{
  "receiverId": 3,
  "productId": 2,
  "content": "商品库存不足，请尽快补货。"
}
```

### 10.4 供应商回复补货提醒

- **POST** `/api/message/restock-reply`

> 当前接口主要供后续微信小程序供应商端使用，必须回复一条自己收到的 `RESTOCK_NOTICE` 消息。

```json
{
  "replyToMessageId": 12,
  "content": "已收到，今天 15:00 到店里补货。"
}
```

### 10.5 标记单条消息已读

- **PUT** `/api/message/read/{id}`

### 10.6 标记全部消息已读

- **PUT** `/api/message/read-all?receiverId={receiverId}`

### 10.7 查询未读消息数

- **GET** `/api/message/unread-count?receiverId={receiverId}`

**响应 data：** `long`

## 11. 系统管理 `/api/system`

### 11.1 分页查询用户

- **GET** `/api/system/user/page`

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| pageNum | int | 否 | 默认 1 |
| pageSize | int | 否 | 默认 10 |
| username | string | 否 | 用户名（模糊） |
| role | string | 否 | 角色：ADMIN / BUSINESS / SUPPLIER |

### 11.2 查询用户详情

- **GET** `/api/system/user/{id}`

### 11.3 创建用户

- **POST** `/api/system/user`

```json
{
  "username": "zhangsan",
  "password": "123456",
  "nickname": "张三",
  "phone": "13800138000",
  "role": "BUSINESS"
}
```

### 11.4 修改用户

- **PUT** `/api/system/user`

> 不允许通过此接口修改密码

```json
{
  "id": 2,
  "nickname": "张三改名",
  "phone": "13900139000"
}
```

### 11.5 删除用户

- **DELETE** `/api/system/user/{id}`

### 11.6 重置密码

- **PUT** `/api/system/user/reset-password/{id}?newPassword=abc123`

### 11.7 数据库备份

- **GET** `/api/system/db/backup`

**说明：** 服务端根据 `hardware.database.backup.mode` 配置执行备份。  
`local` 模式直接调用本机 `mysqldump`，`docker` 模式调用容器内命令。

**响应：** 直接下载 `.sql` 文件（`Content-Disposition: attachment`）

### 11.8 数据库还原

- **POST** `/api/system/db/restore`

**Content-Type：** `multipart/form-data`

**说明：** 服务端根据 `hardware.database.backup.mode` 配置执行还原。  
`local` 模式直接调用本机 `mysql`，`docker` 模式调用容器内命令。

| 参数 | 类型 | 说明 |
|------|------|------|
| file | File | `.sql` 备份文件 |
