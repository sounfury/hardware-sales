# 五金销售管理系统 - 需求文档

## 1. 项目概述

基于 SpringBoot + SA-Token + MySQL 的五金销售管理系统，实现商品进销存管理、供应商管理、交易记录和基础财务管理。

**技术栈：**
- 后端：Java 17 + SpringBoot 3 + SA-Token + MyBatis-Plus + MPJoin（联表查询） + MySQL 8
- 管理端前端：Vue 3
- 供应商端：微信小程序（uni-app）

## 2. 角色定义

| 角色       | 说明                                                 | 使用端       |
| ---------- | ---------------------------------------------------- | ------------ |
| 业务管理员 | 商品管理、进销存操作、交易管理、供应商审核、系统维护、用户管理、数据备份还原 | Web 管理后台 |
| 供应商     | 注册账号、维护自身及产品信息、查看相关交易记录        | 微信小程序   |

## 3. 功能模块

### 3.1 商品信息管理（管理端）

- 商品录入：名称、品牌、规格型号、单位、分类、价格（进价/售价）、库存数量
- 商品列表：分页查询、按名称/品牌/分类筛选
- 商品编辑/删除

### 3.2 供应商管理

#### 小程序端（供应商操作）
- 供应商注册：提交企业名称、联系人、电话、地址、经营范围等
- 供应商信息维护：编辑自身信息
- 产品信息维护：维护自己可供应的商品列表及报价
- 查看供货提醒及反馈
- 查看与自己相关的交易记录（进货单）

#### 管理端（管理员操作）
- 供应商列表查询
- 供应商信息审核：验证供应商信息，通过/驳回
- 向供应商发送供货提醒及反馈（站内消息，小程序端可见）

### 3.3 进销存管理（管理端）

#### 入库（采购进货）
- 录入进货单：选择供应商、商品、数量、进价、日期
- 入库后自动增加库存
- 进货记录查询：按时间、供应商、商品筛选

#### 出库（销售）
- 录入销售单：选择商品、数量、售价、客户信息、日期
- 出库后自动扣减库存
- 销售记录查询：按时间、商品、客户筛选

#### 库存查询
- 当前库存列表
- 进出库明细流水

### 3.4 交易与财务管理（管理端）

- 收入记录：每笔销售自动生成收入记录
- 支出记录：每笔采购自动生成支出记录
- 客户应收款查询：未结算的销售款项
- 供应商应付款查询：未结算的采购款项

### 3.5 系统管理（管理端）

- 用户管理：账号的增删改查、角色分配
- 数据库备份与还原（mysqldump）

## 4. 数据库设计（核心表）

```
user              -- 用户表（管理员、供应商账号）
role              -- 角色表
product_category  -- 商品分类
product           -- 商品表
supplier          -- 供应商信息表
supplier_product  -- 供应商-商品关联表（报价）
purchase_order    -- 采购进货单
purchase_item     -- 采购明细
sales_order       -- 销售出货单
sales_item        -- 销售明细
inventory_log     -- 库存变动流水
finance_record    -- 收支记录
message           -- 站内消息（供货提醒/反馈）
```

## 5. 接口规划（RESTful）

后端统一提供 API，管理端和小程序端共用同一套后端。

```
/api/auth/**           -- 登录注册（含小程序微信登录）
/api/product/**        -- 商品管理
/api/category/**       -- 分类管理
/api/supplier/**       -- 供应商管理
/api/purchase/**       -- 采购进货
/api/sales/**          -- 销售出货
/api/inventory/**      -- 库存查询
/api/finance/**        -- 财务收支
/api/message/**        -- 站内消息
/api/system/**         -- 系统管理（备份、用户管理）
```
