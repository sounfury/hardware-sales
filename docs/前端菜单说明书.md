# 前端菜单说明书

## 1. 使用范围

本文用于讲解管理端左侧菜单的每一个功能点，适合课程演示、答辩讲解和项目交付说明。

当前演示环境建议使用：

- 管理员账号：`admin`
- 管理员密码：`admin123`

说明：

- 当前后台 Web 仅允许管理员登录。
- 供应商账号后续只用于微信小程序端，不再进入这套后台页面。

当前演示数据里可以直接使用的典型对象：

- 供应商：华南电动工具供应中心，联系人是林锋
- 供应商：宏达紧固水暖有限公司，联系人是陈波
- 供应商：安泰电工材料有限公司，联系人是吴倩
- 供应商：联兴综合五金商贸，联系人是赵峰
- 低库存商品：电钻、角磨机、BV2.5 电线、PPR 弯头、切割片

## 2. 推荐先讲的主链路

如果只演示一条主业务链，建议按下面顺序讲：

1. 首页看到电钻库存偏低
2. 在商品列表或首页点“查看供应商”
3. 看到“华南电动工具供应中心”可以供电钻，供货价是 `286.00`
4. 点“提醒补货”，告诉林锋“电钻库存偏低，请尽快安排补货”，商品状态变成“补货中”
5. 进入站内消息，看到补货提醒和供应商反馈，例如“已收到，今天 15:00 去店里补货”
6. 收到货后，在商品列表或首页点“完成补货”，让商品恢复为正常状态
7. 进入采购管理，选择“华南电动工具供应中心”
8. 系统只允许选择这个供应商已经维护过的商品，比如电钻、角磨机、切割片
9. 采购价自动带出 `供应商商品` 里维护的供货价，用户不能手改
10. 提交采购单后，再去库存流水和财务管理看联动结果

这条链路想表达的是：

`供应商商品` 负责回答“谁能供什么、多少钱”，`提醒补货` 负责回答“有没有通知到位”，`采购管理` 负责回答“这次到底向谁买了什么”。

## 3. 菜单逐项说明

### 3.1 首页

页面作用：

- 展示经营总览、低库存商品、最近采购、最近销售、最近消息

推荐讲法：

- 系统管理员一登录首页，就能先看到总收入、总支出、净利润、应收、应付。
- 如果电钻、角磨机、切割片库存偏低，首页会直接列出来，不需要再去商品列表翻找。
- 比如电钻库存只剩 6 台，管理员可以直接点“查看供应商”或“提醒补货”。
- 如果已经给林锋发过补货提醒，首页会把电钻显示成“补货中”，管理员可以继续跟踪，等收货后再点“完成补货”。
- 这时系统会列出谁能供电钻，例如华南电动工具供应中心和联兴综合五金商贸。

可以演示的功能点：

- 查看经营总览卡片
- 查看低库存商品
- 直接跳转“新建采购单”“新建销售单”“商品管理”“用户管理”
- 从低库存商品直接查看供应商报价
- 从低库存商品直接发送补货提醒
- 对补货中的商品执行“完成补货”
- 查看最近采购、最近销售、最近消息

### 3.2 商品管理 > 商品分类

页面作用：

- 维护商品分类，例如电动工具、手动工具、水暖配件、电工电料

推荐讲法：

- 业务管理员先把商品分成“电动工具”“紧固件”“水暖配件”等类别，后面的商品录入才有归属。
- 这一页回答的是“系统里的商品按什么口径分类”。

可以演示的功能点：

- 查看分类列表
- 新增分类
- 编辑分类名称和排序号
- 删除分类

### 3.3 商品管理 > 商品列表

页面作用：

- 维护商品基础信息、参考采购价、销售价、当前库存和补货阈值

推荐讲法：

- 比如系统里有“电钻”“角磨机”“BV2.5 电线”等商品，管理员在这里维护它们的品牌、规格、单位、库存和基础价格。
- 这里的 `采购价` 是商品自身的参考进价，不是具体某次采购的最终成交价。
- 管理员还可以给每个商品单独设置补货阈值，比如电钻阈值设成 `8`，切割片阈值设成 `5`。
- 当电钻库存低于阈值时，页面会给出低库存提示，管理员可以直接点“查看供应商”或“提醒补货”。
- 一旦发了提醒，商品状态会切换成“补货中”；等供应商补到位后，再由管理员手动点“完成补货”。
- 也就是说，这一页先回答“店里卖什么、库存还剩多少”，再把人引到后续补货动作。

可以演示的功能点：

- 按名称、品牌、分类搜索商品
- 新增商品
- 编辑商品
- 删除商品
- 设置补货阈值
- 识别低库存商品
- 低库存商品查看供应商报价
- 低库存商品发送补货提醒
- 对补货中的商品执行完成补货

### 3.4 供应商管理 > 供应商列表

页面作用：

- 维护供应商资料并进行审核

推荐讲法：

- 这里管理的是供应商主体信息，比如企业名称、联系人、联系电话、经营范围。
- 比如“华南电动工具供应中心”的联系人是林锋，“宏达紧固水暖有限公司”的联系人是陈波。
- 只有审核通过的供应商，后面才能进入“供应商商品”“站内消息”“采购管理”这条链路。
- 所以这页回答的是“这个供应商能不能正式参与业务”。

可以演示的功能点：

- 按企业名称、审核状态筛选
- 新增供应商档案
- 编辑供应商档案
- 删除供应商
- 审核通过或驳回供应商

### 3.5 供应商管理 > 供应商商品

页面作用：

- 维护“供应商和系统商品之间的真实供货关系”

推荐讲法：

- 这页不是下采购单，它是先维护一份供货清单和报价清单。
- 比如华南电动工具供应中心维护了“电钻、角磨机、切割片”的供货关系，电钻供货价是 `286.00`。
- 联兴综合五金商贸也能供电钻，但报价是 `292.00`。
- 所以这页回答的是“谁能供这个商品、供货价是多少”，它是采购前的主数据，不是交易单据。

可以演示的功能点：

- 按供应商和商品筛选供货关系
- 新增供货商品
- 编辑供货价和备注
- 删除供货关系

讲解时可以直接强调：

- `供应商商品` 不和 `采购单` 冲突
- `供应商商品` 是报价和能力清单
- `采购单` 是实际成交记录

### 3.6 供应商管理 > 站内消息

页面作用：

- 管理普通消息、补货提醒消息和供应商补货回复

推荐讲法：

- 当管理员发现电钻库存偏低，可以给林锋发一条补货提醒。
- 当前演示数据里，林锋已经回复了“已收到，今天 15:00 去店里补货”，这时管理员在后台就能看到这条回复。
- 所以这页回答的是“有没有把需求发给供应商，以及供应商有没有回应”。

可以演示的功能点：

- 查看未读数
- 按已读状态、消息类型筛选
- 发送普通消息
- 查看补货提醒消息
- 查看补货回复消息和回复来源
- 标记单条已读
- 全部标记已读
- 轮询刷新未读数量

### 3.7 进销存 > 采购管理

页面作用：

- 创建采购单、查看采购详情、跟踪付款结算

推荐讲法：

- 这一页已经和 `供应商商品` 打通了。
- 比如管理员要向华南电动工具供应中心采购，先选供应商，系统只会显示它已经维护过的商品，比如电钻、角磨机、切割片。
- 用户这里只需要选商品和数量，价格不会让用户自己改。
- 电钻的采购价会自动取华南电动工具供应中心在 `供应商商品` 页面维护的 `286.00`。
- 这样就能清楚说明：报价在前面维护，采购单在这里正式落地。

可以演示的功能点：

- 按采购单号、供应商、日期筛选采购单
- 新建采购单
- 采购单自动计算总金额
- 采购单只允许选择该供应商的供货商品
- 采购价自动复用供应商商品报价
- 查看采购单详情
- 结算采购单

这页最适合直接解释你刚才提的那个问题：

- `新增供货商品` 是先维护报价
- `提醒补货` 是先发业务消息
- `采购单` 是最后正式下单

### 3.8 进销存 > 销售管理

页面作用：

- 创建销售单、查看销售详情、跟踪客户结算

推荐讲法：

- 比如给“恒安物业”或者“鑫达水电”出货，业务管理员在这里登记客户、销售日期、销售商品、数量和售价。
- 销售单创建以后，库存会自动减少，财务里会生成收入记录。
- 所以这页回答的是“货卖给谁了、卖了多少、钱收没收”。

可以演示的功能点：

- 按销售单号、客户名称、日期筛选
- 新建销售单
- 自动计算销售总金额
- 查看销售单详情
- 结算销售单

### 3.9 进销存 > 库存流水

页面作用：

- 查看每个商品的入库和出库变化

推荐讲法：

- 采购单创建后会有入库记录，销售单创建后会有出库记录。
- 比如电钻采购入库 2 台后，库存会增加；卖出后又会减少，这些都会在这里留痕。
- 所以这页回答的是“库存为什么变了，变动是从哪张业务单来的”。

可以演示的功能点：

- 按商品筛选
- 按出入库类型筛选
- 按日期区间筛选
- 查看变动前库存、变动后库存
- 查看关联业务类型是采购还是销售

### 3.10 财务管理

页面作用：

- 查看收入、支出、应收、应付和财务明细

推荐讲法：

- 采购单会生成支出记录，销售单会生成收入记录。
- 例如采购华南电动工具供应中心的电钻会形成一笔应付，销售给客户会形成一笔应收。
- 所以这页回答的是“业务做完以后，钱的流向有没有被系统记录下来”。

可以演示的功能点：

- 查看总收入、总支出、净利润、应收、应付
- 按收入/支出筛选
- 按结算状态筛选
- 按日期筛选
- 查看财务明细关联的是采购单还是销售单

### 3.11 系统管理 > 用户管理

页面作用：

- 管理管理员、业务员、供应商账号

推荐讲法：

- 这里维护的是登录账号，不是业务资料本身。
- 比如 `admin` 是系统管理员，`business01` 是业务管理员，`supplier_tool_01` 是供应商账号。
- 供应商账号会再和“供应商列表”里的企业资料绑定起来。
- 所以这页回答的是“谁能登录系统、登录后是什么角色”。

可以演示的功能点：

- 按用户名、角色筛选
- 新增用户
- 编辑用户
- 删除用户
- 重置密码
- 区分系统管理员、业务管理员、供应商角色

### 3.12 系统管理 > 数据库备份

页面作用：

- 管理员手动导出当前数据库 SQL 备份，或上传备份文件执行还原

推荐讲法：

- 管理员在这里可以手动做数据库备份，系统会按当前后端数据库配置导出一份 `.sql` 文件。
- 如果需要恢复演示环境，也可以把之前导出的 `.sql` 文件重新上传还原。
- 这个功能更偏运维保障，不是日常业务主链路，所以演示时点到为止即可。

可以演示的功能点：

- 点击下载数据库备份
- 上传 `.sql` 文件执行数据库还原
- 说明该功能仅限管理员使用

## 4. 一句话区分最容易混淆的三个页面

如果老师追问“供应商商品、补货提醒、采购单有什么区别”，可以直接这样回答：

- `供应商商品`：维护谁能供什么、供货价是多少
- `补货提醒`：库存低时先通知供应商准备补货
- `采购管理`：最终正式生成采购单、更新库存、形成财务记录

## 5. 推荐结束语

这个系统不是只做静态信息录入，而是已经形成了一条完整业务链：

商品库存异常后，系统能找到对应供应商，能发补货提醒，能根据供应商报价创建采购单，最后把库存和财务一起联动起来。
