# 微信小程序新增功能接口文档

> 适用范围：仅包含“客户下单简化版”本次新增或改动到的小程序接口，不包含原有全量接口。
>
> Base URL: `http://localhost:8080`
>
> 认证方式：登录成功后在请求头中携带 `Authorization: <token>`

## 1. 统一响应格式

```json
{
  "code": 200,
  "msg": "success",
  "data": {}
}
```

| code | 说明 |
|------|------|
| 200 | 成功 |
| 401 | 未登录或登录已过期 |
| 500 | 业务异常，具体原因见 `msg` |

## 2. 本次新增能力范围

本次主要为微信小程序新增“客户角色下单”能力，涉及三类接口：

- 小程序注册登录：支持客户注册、客户登录
- 商品浏览：客户可分页查看商品、查看商品详情
- 客户预定：立即预定、我的预定分页、我的预定详情

## 3. 角色约定

| 角色值 | 说明 |
|--------|------|
| `SUPPLIER` | 供应商 |
| `CUSTOMER` | 客户 |
| `""` | 已注册但尚未完善供应商资料的用户 |

小程序登录成功后前端建议按角色跳转：

- `SUPPLIER`：进入供应商功能页
- `CUSTOMER`：进入客户商品列表页
- `""`：进入供应商资料完善页

## 4. 认证接口 `/api/auth`

### 4.1 小程序注册

- **POST** `/api/auth/miniapp/register`
- **无需认证**

说明：

- 本次新增字段 `registerType`
- `SUPPLIER` 注册后角色为空字符串
- `CUSTOMER` 注册后角色直接为 `CUSTOMER`

请求体：

```json
{
  "registerType": "CUSTOMER",
  "username": "customer01",
  "password": "123456",
  "nickname": "张三",
  "phone": "13800138000"
}
```

`registerType` 可选值：

- `SUPPLIER`
- `CUSTOMER`

响应 data：

```json
{
  "token": "xxxx-xxxx-xxxx",
  "user": {
    "id": 21,
    "username": "customer01",
    "nickname": "张三",
    "avatar": null,
    "phone": "13800138000",
    "role": "CUSTOMER",
    "status": 1,
    "createTime": "2026-05-15T15:00:00",
    "updateTime": "2026-05-15T15:00:00"
  }
}
```

- **无需认证**

说明：

- 本次登录接口已放行 `CUSTOMER`
- 允许进入小程序的角色为：`SUPPLIER`、`CUSTOMER`、空角色

请求体：

```json

  "password": "123456"
}
```

响应 data：

## 5. 商品浏览接口 `/api/product`

说明：

- 本次改动后，`CUSTOMER` 可以调用商品查询接口
- 客户只读，不允许新增、修改、删除商品

### 5.1 商品分页

- **GET** `/api/product/page`
- **需要登录**

请求参数：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| pageNum | int | 否 | 默认 1 |
| pageSize | int | 否 | 默认 10 |
| name | string | 否 | 商品名称模糊查询 |
| brand | string | 否 | 品牌模糊查询 |
| categoryId | long | 否 | 分类 ID |

示例：

```text
/api/product/page?pageNum=1&pageSize=10&name=电钻
```

响应 data：

```json
{
  "records": [
    {
      "id": 1,
      "categoryId": 1,
      "name": "电钻",
      "brand": "博世",
      "spec": "GSB 13 RE",
      "description": "常用于墙面打孔和基础安装作业",
      "unit": "台",
      "purchasePrice": 335.00,
      "salePrice": 499.00,
      "stock": 6,
```

## 6. 客户预定接口 `/api/sales/miniapp`

说明：

- 本组接口仅允许 `CUSTOMER` 调用
- 后端会自动识别当前登录客户
- 后端会自动读取商品当前 `salePrice`
- 预定成功后会自动生成一张 `未结算` 销售单

### 6.1 立即预定

- **POST** `/api/sales/miniapp/preorder`
- **需要登录**
- **仅 `CUSTOMER` 可调用**

请求体：

```json
{
  "items": [
    {
      "productId": 1,
      "quantity": 2
    }
  ],
  "remark": "到店自提"
}
```

字段说明：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| items | array | 是 | 预定明细 |
| items[].productId | long | 是 | 商品 ID |
| items[].quantity | int | 是 | 购买数量，必须大于 0 |
| remark | string | 否 | 订单备注 |

响应 data：

```json
{
  "id": 8,
  "orderNo": "SO202605151556451410",
  "paymentStatus": 0
}
```

后端处理规则：

- 自动写入 `customerUserId`
- 自动写入 `customerName`、`customerPhone`
- 自动写入 `orderSource = MINIAPP`
- 自动计算总金额
- 自动扣减库存
- 自动生成财务收入记录

### 6.2 我的预定分页

- **GET** `/api/sales/miniapp/my-page`
- **需要登录**

## 7. 小程序前端建议最小接入顺序

1. 注册页增加 `registerType`
2. 登录后按 `user.role` 分流页面
3. 客户首页接 `/api/product/page`
4. 商品详情或弹窗接 `/api/product/{id}`
5. 立即预定接 `/api/sales/miniapp/preorder`
6. 我的预定页接 `/api/sales/miniapp/my-page`
7. 订单详情页接 `/api/sales/miniapp/{id}`
